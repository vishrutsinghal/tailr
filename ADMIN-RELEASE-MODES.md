# TailTrail Admin Release Modes

This file is for TailTrail maintainers only. It is excluded from internal distributions.

TailTrail has one admin source tree and two distribution modes:

- `internal`: company/private usage package with no public release surface.
- `public`: public/open-market package with public license, contribution, security, claim, and release-check files.

Normal internal users should not see public release choices, public release commands, public licensing workflow, or switching instructions. They should receive only an internal distribution generated by an admin.

## Rules

- Core feature changes start in the admin source tree.
- Feature changes that are safe for both audiences are exported to both distributions.
- Public-only release/legal/contribution/claim changes stay in the public distribution path.
- Internal-only policies, examples, rollout notes, and org-specific workflows stay out of the public distribution.
- Do not manually maintain two unrelated codebases.

## Admin Export Commands

Preview internal files:

```bash
python3 scripts/export-release.py --mode internal --target /tmp/tailtrail-internal --list
```

Export internal package:

```bash
python3 scripts/export-release.py --mode internal --target /tmp/tailtrail-internal --force
```

Admin wrapper:

```bash
TAILTRAIL_ADMIN=1 python3 scripts/tailtrail.py admin export --mode internal --target /tmp/tailtrail-internal --force
```

Preview public files:

```bash
python3 scripts/export-release.py --mode public --target /tmp/tailtrail-public --list
```

Export public package:

```bash
python3 scripts/export-release.py --mode public --target /tmp/tailtrail-public --force
```

Admin wrapper:

```bash
TAILTRAIL_ADMIN=1 python3 scripts/tailtrail.py admin export --mode public --target /tmp/tailtrail-public --force
```

## Internal Distribution Behavior

Internal export excludes:

- `LICENSE`
- `SECURITY.md`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `PUBLIC-CLAIMS.md`
- `RELEASE-CHECKLIST.md`
- `.github/workflows/tailtrail-ci.yml`
- `scripts/release-check.py`
- public/admin planning docs such as roadmap and honest review files
- this admin release-mode file
- the export script itself

Internal export writes `.tailtrail-internal-release` and does not write `.tailtrail-public-release`.

Because the public marker is absent, the normal `tailtrail.py` command surface does not show or allow `release-check`.

## Public Distribution Behavior

Public export includes the public release files and writes `.tailtrail-public-release`.

Because the public marker is present, `tailtrail.py release-check` is available in the public distribution. Admin source trees can also run it with:

```bash
TAILTRAIL_ADMIN=1 python3 scripts/tailtrail.py release-check
```

## Validation

Admin source validation:

```bash
python3 scripts/check-tailtrail.py
TAILTRAIL_ADMIN=1 python3 scripts/tailtrail.py release-check
python3 scripts/tailtrail.py doctor
```

Internal distribution smoke check:

```bash
python3 /tmp/tailtrail-internal/scripts/tailtrail.py hello
python3 /tmp/tailtrail-internal/scripts/tailtrail.py help
python3 /tmp/tailtrail-internal/scripts/tailtrail.py doctor
```

Public distribution smoke check:

```bash
python3 /tmp/tailtrail-public/scripts/tailtrail.py hello
python3 /tmp/tailtrail-public/scripts/tailtrail.py help
python3 /tmp/tailtrail-public/scripts/tailtrail.py release-check
```

## Why This Exists

Internal users need the TailTrail product without public-release clutter. Public users need license, security, contribution, and claim hygiene. Admins need a reliable way to produce both without drifting two separate codebases.
