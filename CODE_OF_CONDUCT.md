# Code Of Conduct

TailTrail follows a simple contributor standard: be respectful, specific, and constructive.

## Expected Behavior

- Use welcoming and professional language.
- Focus critique on code, docs, behavior, and evidence.
- Respect different experience levels and working contexts.
- Keep security reports private until maintainers can respond safely.
- Avoid sharing secrets, private logs, customer data, PII, PHI, or proprietary code in public discussions.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private security details before coordinated disclosure.
- Sharing credentials, tokens, customer data, private logs, or proprietary code.
- Repeatedly derailing technical discussions after maintainers ask to refocus.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block accounts; or decline contributions that violate this standard.
